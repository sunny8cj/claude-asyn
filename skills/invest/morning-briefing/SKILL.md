### 3.3 消息影响分析格式

**影响程度：** 短期（1-3天）/ 中期（1-4周）/ 长期（1月+）

**辩证分析：**

对每只持仓/自选股，搜索：

- `{公司名} 今日 行情 {当前日期}` — 用API获取前一日收盘价验证

### 4.3 个股数据交叉验证（强制步骤）

**问题根源：** WebSearch 搜索结果会混杂不同时期的数据。例如 "连续涨超20%、异常波动公告" 等新闻标题可能来自几周前甚至几个月前，但搜索引擎会返回在结果中，极易误用。

**验证流程：**

1. **第一步：用搜索引擎获取概览信息（新闻、公告）**
   - 搜索：`"{公司名}" "{当前日期}"` 或 `"{公司名}" 公告 {当前日期}`
   - 记录新闻/公告的标题和关键词

2. **第二步：用API获取精确收盘价（必须执行）**
   ```bash
   # A股（腾讯行情API，返回最新收盘价）
   curl "http://qt.gtimg.cn/q=sh{去掉后缀的代码}"   # 上交所，如 sh600089
   curl "http://qt.gtimg.cn/q=sz{去掉后缀的代码}"   # 深交所，如 sz300274
   
   # 港股（腾讯行情API）
   curl "http://qt.gtimg.cn/q=hk{去掉后缀的代码}"   # 港股，如 hk00883
   
   # API返回格式示例：v_sz300274="1~阳光电源~300274~139.85~139.50~142.99~138.00~..."
   # 第4个字段=最新价，第5个字段=昨收，第6个字段=今开，第7个字段=最高，第8个字段=最低
   # 涨跌幅 = (最新价 - 昨收) / 昨收 * 100
   ```

3. **第三步：搜索结果与API数据一致性检查**
   - 将搜索结果中的股价数字与API返回的最新价对比
   - **差异 > 2% 时，以API数据为准**，搜索结果的股价视为过时
   - 搜索结果中的新闻/公告标题，必须确认其日期标注为当前日期或前一日，才能引用

4. **第四步：周涨跌验证**
   - 在输出中描述"本周/上周"走势时，必须用API数据对比本周一开盘价与最新价
   - **禁止**仅凭搜索结果中的新闻标题（如"连续大涨""连续下跌"）判断周涨跌
   - 搜索结果中的"连涨N天""异常波动公告"等描述，必须确认其对应的时间段是当前周，才能引用

**示例错误与修正：**
```
错误：搜索到"大金重工连续涨超20%、发布异常波动公告"，误认为本周上涨
实际：该公告来自4月底，5月5-8日当周大金重工实际是下跌的（从84.00跌至约82.98）
修正方法：用腾讯API查询5月8日收盘价，与5月5日收盘价对比，确认实际涨跌方向
```

## 数据交叉验证铁律

| 数据类型 | 获取方式 | 验证方式 | 差异阈值 |
|---------|---------|---------|---------|
| A股股价 | 腾讯API `qt.gtimg.cn` | 东方财富API或搜索 | > 2% |
| 港股股价 | 腾讯API `qt.gtimg.cn` | 富途/英为财情 | > 2% |
| 美股股价 | 搜索引擎（2+来源） | Yahoo Finance | > 2% |
| 大宗/期货 | 东方财富API | 搜索 | > 3% |
| 消息/新闻 | WebSearch | 至少2个独立来源 | - |

**禁止行为：**
- 凭印象或记忆引用时效性数据
- 不标注数据日期和来源
- 使用过时的历史数据冒充最新数据
- **禁止仅凭搜索结果的新闻标题判断个股涨跌方向**（搜索结果中可能混杂不同时期的数据）
- **禁止在引用新闻/公告时不确认其日期标注**
- **禁止在输出中写"连涨N天""连续下跌"等描述时不核对API数据**

**强制验证流程：**
1. 个股价格必须先通过腾讯API获取，再用搜索引擎补充新闻/公告
2. 搜索结果中的任何价格数字必须与API返回的最新价对比，差异>2%视为过时
3. 搜索结果中的新闻标题（特别是"异常波动""连涨/连跌"类）必须确认其日期为当前周
4. 输出前，对所有持仓/自选股的涨跌方向做最终核对
