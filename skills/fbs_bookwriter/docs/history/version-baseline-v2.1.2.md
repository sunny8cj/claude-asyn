# 版本基线归纳（以 v2.1.2 为当前基线）

> 目的：给交付、审核、维护提供单一版本口径。  
> 原则：以 **v2.1.2** 为唯一当前基线；既往版本仅保留「能力演进摘要」，不再作为运行时决策依据。  
> **上一版能力归纳**（v2.1.1 正式发布线）：[`version-baseline-v2.1.1.md`](./version-baseline-v2.1.1.md)

## 当前基线（必须遵循）

- 当前有效版本：`2.1.2`
- 当前 `plugin-id`：`fbs-bookwriter-v212`（见 `SKILL.md` frontmatter）
- 当前入口基线：`SKILL.md`、`openclaw/fbs_bookwriter/SKILL.md`、`.codebuddy/agents/fbs-team-lead.md`
- 当前机读基线：`fbs-runtime-hints.json`（`version` 字段）、`_plugin_meta.json`
- 当前门禁基线：
  - `s0-exit-gate`
  - `expansion-gate`
  - `polish-gate`
  - `release-governor`
  - `final-manuscript-clean-gate`
- 当前交付约束：全稿/终稿/终审稿不得包含写作过程标注

## 既往版本归纳（按能力线）

| 版本 | 定位 | 说明 |
|------|------|------|
| `1.0` | 初版可用 | 建立基础写作与 Markdown 交付能力 |
| `1.60` | 预规模化 | 引入场景包雏形与评分体系 |
| `2.0` | 工作流重构 | S0-S6 主流程、S/P/C/B 质检主框架形成 |
| `2.0.1` | 工程对齐 | 宿主适配与版本口径规范化 |
| `2.0.2` | 宿主深度适配 | 智能记忆、能力探测、会话快照等关键能力落地 |
| `2.0.3`（内部线） | 内部能力迭代 | 已并入后续正式线，不单独作为发布基线 |
| `3.0-internal-track`（内部线） | 插件总线探索 | 已并回 `2.1.0+`，不单独发布 |
| `2.1.0` | 收口升级 | 入口一致化、恢复体验、脚本桥接与审计链成型 |
| `2.1.1` | 质量与触发治理 | 门禁强绑定、终稿治理、终稿洁净、发布链闭环；7 锁与阶段推进执行锁 |
| `2.1.2` | 写作优先与工程治理 | 在 2.1.1 之上强化 Writing-First、宿主契约、打包与可观测性、插件路由 KPI 与文档基线统一 |

## 使用规则（防歧义）

- 对外说明版本时，只说：**「当前基线为 2.1.2」**。
- 讨论 **2.1.1** 时，只引用 [`version-baseline-v2.1.1.md`](./version-baseline-v2.1.1.md) 或 `CHANGELOG` 的 `[2.1.1]` 节，不展开内部迭代流水。
- 任何脚本/文档行为冲突，以 **2.1.2** 入口文件和门禁脚本为准。
- 旧版本条目用于追溯，不可作为「跳过当前门禁」的依据。
